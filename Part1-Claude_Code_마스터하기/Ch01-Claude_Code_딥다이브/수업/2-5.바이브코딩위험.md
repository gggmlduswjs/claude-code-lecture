### 시나리오 ① 인증/권한

- **API 엔드포인트에 인증 미들웨어 누락** — `GET /api/users`가 인증 없이 노출되면 URL만 알면 전체 사용자 조회 가능 (현관문 비밀번호 안 걸어둔 상태)
- **관리자 API에 권한 체크 누락** — 일반 사용자가 `/api/admin/delete-user` 호출
- **JWT 토큰 검증을 클라이언트에서만 수행** — 서버 직접 호출로 우회됨
- **Row Level Security 누락 (Supabase)** — 한 토큰으로 남의 데이터까지 조회

> 이런 코드는 **"동작"은 합니다.** 정상 사용자가 정상 흐름으로 쓰면 무사. 단, **문이 활짝 열린 상태**일 뿐 — 누가 일부러 열어보기 전까지 안 보입니다.
> 

### 시나리오 ② 데이터 처리 — SQL Injection / XSS / CSRF

```tsx
// AI가 생성한 코드 — 위험!
const query = `SELECT * FROM users WHERE email = '${userInput}'`;

// 안전한 버전
const query = `SELECT * FROM users WHERE email = $1`;
const result = await db.query(query, [userInput]);
```

AI는 종종 문자열 보간 패턴을 생성합니다. **코드를 안 읽으면 이 차이를 알 수 없습니다.**

### 시나리오 ③ 성능과 확장성

- **N+1 쿼리** — 100명 사용자에 100번 DB 호출
- **메모리 누수** — 이벤트 리스너 해제 누락
- **동시성 문제** — 재고 차감 race condition
- **무한 루프** — 엣지 케이스 재귀
