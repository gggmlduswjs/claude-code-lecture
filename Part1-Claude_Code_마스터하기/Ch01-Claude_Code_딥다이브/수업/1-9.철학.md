풍부한 입력 채널: @로 파일 참조 / 이미지 드래그 / URL (/permissions 화이트리스트) / cat error.log | claude.


Claude가 인터뷰하게 하기 — "AskUserQuestion 도구로 자세히 인터뷰해줘. 명백한 질문 X — 본인이 고려 못한 어려운 부분 파고들기. 모든 것 다룰 때까지 → SPEC.md에 사양 작성." 사양 완료 후 새 세션에서 깨끗한 컨텍스트로 구현.

4. 비대화형 모드 - claude -p, --bare

# 일회성
claude -p "이 프로젝트가 뭐 하는지 설명"
-> 클로드 세션을 열지 않고 대화를 할 수 있다


# 구조화 출력
claude -p "모든 API 엔드포인트 나열" --output-format json

# 빠른 시작 — hooks/skills/plugins/MCP/CLAUDE.md 모두 건너뜀
claude --bare -p "이 파일 요약" --allowedTools "Read"

# CI/CD에서 자율
claude --permission-mode auto -p "fix all lint errors"

--bare = 시작 빠름 + 재현성. CI·pre-commit hook에 그대로 활용.
