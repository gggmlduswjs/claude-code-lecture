스킬의 특이한 로딩 방식

신입사원에게 매번 말로 지시하는 대신 매뉴얼 한 부 주는 것. SKILL.md에 절차/형식/체크리스트 → 매번 같은 품질.

2단계 로딩 — 평소 description만 (~100B), 호출/매칭 시 본문 + 지원 파일 풀 로드.


.claude/skills/
└── ppt-generator/
    ├── SKILL.md          ← 핵심 (필수, 대문자!)
    ├── template.md
    ├── examples/
    └── scripts/

## 절차
1. 발표 주제, 대상 청중, 발표 시간 확인
2. 목차 + 슬라이드 구성 설계
3. 각 슬라이드 내용 작성
4. .pptx 출력

- [ ] 슬라이드 수 적절한가?
- [ ] 핵심 메시지 있는가?

[mcp vs cli] 

mcp-하나의 서비스를 패키지로 담아둠
cli-리모컨

**CLI 추세 다섯 이유** — ① 토큰 0 / ② 재현성 / ③ CI 재사용 / ④ 버전 관리 (git diff) / ⑤ 컨텍스트 깨끗 (큰 출력은 파일).

**원칙: "기본은 CLI, 예외만 MCP"** — Notion/Linear MCP 하나가 3~10K씩 깎아먹는다.


[hooks]

훅 : 이벤트를 자동으로 실행해주는것 -> 이 규칙을 따라야 돼

PreToolUse	도구 호출 직전	입력 검증, 위험 명령 차단
PostToolUse	도구 호출 직후	후처리 (lint, format)
Notification	응답 대기	알림 (terminal-notifier)
Stop	턴 종료	보고서, 상태 저장





// ~/.claude/settings.json — PostToolUse 자동 포맷
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [{
          "type": "command",
          "command": "npx prettier --write \"$CLAUDE_PROJECT_DIR\" 2>/dev/null || true"
        }]
      }
    ]
  }
}
