탐색 → 계획 → 코드 → 커밋

잘못된 문제를 풀지 않는 4단계 워크플로우
1
⏸ Plan Mode
탐색
파일 읽기, 구조 이해 — 변경 X
> /src/auth를 읽고 세션·로그인 처리 방식 이해
2
⏸ Plan Mode
계획
상세 구현 계획 작성. Ctrl+G로 직접 편집 가능
> OAuth 추가하려고 한다. 어떤 파일 수정? 세션 흐름은?
3
⚡ Auto Mode
구현
계획대로 실행. 검증 단계 포함하면 한 사이클에 끝
> 계획대로 OAuth 구현. 콜백 핸들러 테스트 작성, 통과 확인
4
🟢 Default
커밋
설명적 메시지로 커밋, PR 열기
> 설명적 메시지로 커밋, PR 열기
📝 작업이 작거나(오타·한 줄 수정) diff를 한 문장으로 설명할 수 있으면 — Plan 건너뛰기 OK.
