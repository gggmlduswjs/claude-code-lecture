### 식별 방법 — Probabilistic Stalls를 잡아라

- **반복되는 인간 호출** — 같은 패턴의 "이거 해주세요"
- **긴 idle 시간** — agent가 멈추고 응답을 기다리는 구간
- **에러 후 포기** — agent가 시도 실패 후 인간에 우회 요청


가장 먼저 검토해야할것 cli

## 왜 CLI가 *Primary Bridge*인가

- agent가 이미 텍스트로 작동 → CLI도 텍스트 in/out, 가장 자연스러운 인터페이스
- 모든 시스템이 CLI 제공 — GUI는 그 위 layer
- CLI는 **composable** — `gh pr list | jq` 식 파이프 조합
- 자동화 가능 — 명령 한 줄을 Skill로 바로 codify

- **High-level architectural decisions** = 방향·trade-off·다음 다리 후보 → **human**


사람이 결정하고, 그 결정을 다시 agent가 실행할 수 있는 형태(Skill·CLI·MCP)로 codify → 루프가 닫힌다.
