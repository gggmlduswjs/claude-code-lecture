- **Engineering** = "시스템을 설계하는 행위", 결과물은 **동작하는 구조물**


Karpathy가 직접 정리한 3가지 새 작업:

1. **Extensive System Design** — 광범위한 시스템 설계 (도구·권한·sub-agent 구조)
2. **Intentional Thought** — 즉흥이 아닌 *의도된* 흐름 설계
3. **Construction of Robust Environments** — agent가 *autonomously* 작동할 환경 자체를 만들기


AI 에이전트가 multi-step task를 자율 실행하는 시스템을 
— rigorous context management × recursive validation loops × integrated tool orchestration의 통합을 통해 
— 설계하고 최적화하는 전문 엔지니어링 분과.


- **Rigorous Context Management** — agent의 context를 *fresh + condensed*로 큐레이션 (Pillar I)
- **Recursive Validation Loops** — agent가 자기 작업을 자기가 평가, 실패 시 *재귀적*으로 자기 수정 (Pillar II)
- **Integrated Tool Orchestration** — CLI·API·MCP·Slash가 흔어진 게 아니라 **통합된 시스템**으로 노출 (Pillar III·IV·V)
