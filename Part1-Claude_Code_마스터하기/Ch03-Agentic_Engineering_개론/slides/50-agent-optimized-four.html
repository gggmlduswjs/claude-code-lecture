Optimization Checklist

Agent-Optimized Codebase — 4요소

매분기 audit에 사용할 수 있는 — 진단 프레임워크. 위에 메타 원리로 *Consistency over DRY*가 작동한다
☣️
Element 1
Pattern Contamination 제거
dead code · deprecated logic · 미완성 마이그레이션을 끈질기게 정리. 같은 일을 하는 경쟁 패턴이 발견되는 즉시 — 한 쪽을 표준으로 고정하고 나머지를 제거.
📐
Element 2
Structural Consistency
디렉토리 구조와 네이밍 컨벤션을 rigid·predictable하게 유지. agent가 "어디에 두지?"를 묻지 않고 — 표준 위치를 예측적으로 찾는다.
📊
Element 3
Agent-Specific Logging
agent 소비용 machine-readable 로그 (JSON line). 인간 가독 텍스트가 아니라 — agent가 파싱·쿼리·self-correction에 쓸 수 있는 구조화 로그.
📜
Element 4
Golden Rules + RAG Docs
아키텍처 표준과 opinionated rules를 — repo 안에 agent RAG에 최적화된 포맷으로 인코딩. CLAUDE.md → GOLDEN_RULES.md retrieval.
이 4요소를 모두 만족하면 — agent의 출력 variance가 구조적으로 감소한다
