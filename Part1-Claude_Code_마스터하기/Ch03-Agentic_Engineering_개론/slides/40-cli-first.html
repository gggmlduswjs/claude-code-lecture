자율 실행
no GUI scraping
Reason 1 · 자연스러운 인터페이스
agent가 이미 텍스트로 작동. CLI도 텍스트 in/out — 가장 자연스러운 매칭.
Reason 2 · 보편적 접근
모든 시스템이 CLI를 제공. GUI는 그 위의 layer일 뿐. 핵심 기능 = CLI.
Reason 3 · Composable
gh pr list | jq 식 파이프 조합. agent가 가장 잘하는 영역.
Reason 4 · Codify 용이
명령 한 줄을 Skill로 바로 codify. GUI는 Selenium / Playwright 필요.
새 시스템 도입 시 가장 먼저 묻는 질문: "CLI가 있는가?" — Pillar III의 첫 실천 습관.
