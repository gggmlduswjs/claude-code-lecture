⭐ Core Definition
— Boris Cherny · Claude Code 리드 (Anthropic)
Closed-loop validation은
캐주얼한 prompting과 프로페셔널 엔지니어링을
가르는 1차 differentiator(차별 요인)다.
Self-correction
agent가 스스로 수정한다 — 사람이 끼어들어서 수정하는 게 아니다. 자기 출력의 토큰 분포만으로는 평가가 신뢰 불가.
Loop
한 번이 아니라 반복 구조 — 실패 → 분석 → 수정 → 재시도 → ... 통과할 때까지. 닫힌 루프(closed)여야 사람 개입이 없다.
⚡ "crappy output"과 "validated work"의 차이는 — closed-loop self-correction의 존재 여부로 정의된다
