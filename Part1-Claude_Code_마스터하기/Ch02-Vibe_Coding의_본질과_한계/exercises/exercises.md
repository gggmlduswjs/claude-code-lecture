# Part 1 · Ch02 실습 — Vibe Coding 90분 챌린지

> **이 챕터의 실습 목적**
> Vibe Coding을 책으로만 이해하지 말고, **한 번 끝까지 가본다.** CLAUDE.md, Skill, Hook, 서브에이전트 — Part 1에서 곧 배울 도구들을 **일부러 전부 빼고**, 모델 한 명에게 자연어로만 90분 동안 무언가를 만들어봅니다.
>
> 다음 챕터(Agentic Engineering)에서 **이 결과물에 직접 Agentic 패턴을 입히게** 됩니다. 그때 "이전에는 이렇게 답답했지" 하고 본인 손으로 비교해보는 것이 진짜 목표 — Vibe vs Agentic의 차이를 **체감**으로 갖기 위함입니다.

**예상 소요 시간**: 90분 (작업 60~75분 + 짧은 회고 15~30분)

**준비물**
- Claude Code 또는 Codex가 설치된 본인 머신 (Part 0 Ch02 이후로 갖춰져 있어야 함)
- 빈 디렉토리 1개 (아래에서 만듭니다)

## 🟡 실습 — 90분 안에 아이디어 1개 끝까지 만들어보기

### 무엇을 하는가
**손이 닿는 모든 안전망을 일부러 빼고** AI에게 자연어로만 지시해 작은 무언가를 만들어봅니다. Karpathy의 정의 그대로 — *"라이브러리 이름만 알고 만든다"*.

만든 결과물은 버리지 않습니다. **Ch03(Agentic Engineering)에서 이 결과물에 CLAUDE.md·검증 루프·자동화를 입혀** Vibe → Agentic 전환을 본인 손으로 경험하게 됩니다.

### 사전 규칙 (의도적으로 빼는 것들)

이 90분 동안 다음을 **사용하지 않습니다**:
- ❌ `CLAUDE.md` (아예 만들지 않음)
- ❌ `.claude/settings.json` permissions, hooks
- ❌ Skill, Subagent, MCP
- ❌ Plan Mode (`Shift+Tab`), 4체크
- ❌ 테스트, lint, typecheck 등 검증 도구
- ❌ 라이브러리 공식 문서 미리 읽기

오직:
- ✅ Claude Code 세션 1개 (또는 Codex)
- ✅ 자연어 프롬프트
- ✅ 본인의 직감

### 단계

#### Step 0. 아이디어 1개 정하기 (5분)

다음 조건을 만족하는 작은 아이디어:
- 1~3개 파일로 끝낼 수 있음
- 외부 사용자 없음 (본인만 사용)
- 결제·인증·PII 없음
- 5초 안에 결과가 눈에 보이는 종류

**예시 아이디어**:
- 본인 GitHub PR을 최근 30개 가져와서 작성 시간대 히트맵으로 시각화
- 가계부 CSV를 카테고리별 막대그래프로
- 매일 아침 IT 뉴스 RSS 3개를 한 줄 요약으로 묶어주는 스크립트
- 본인 슬랙 DM 검색을 정규식으로 필터링하는 CLI
- 본인 캘린더 1주일을 ASCII 아트로 출력

> ⚠️ "회사 프로젝트의 X 기능 추가" 같은 건 **금지**. 90분에 안 끝나고, Track 2 영역이라 Vibe로 가면 안 됨.

#### Step 1. 작업 공간 셋업 (5분)
```bash
cd ~/projects   # 또는 본인 작업 디렉토리
mkdir vibe-90min && cd vibe-90min
git init
```

> Claude Code를 띄울 때 `claude` 한 줄. `/init`도 하지 마세요. CLAUDE.md를 빈 디렉토리에 두지 않는 것이 이번 실습의 핵심.

#### Step 2. 60~75분 안에 동작 가능한 상태까지 (60~75분)

다음 규칙을 지키며 자연어로만 진행:

- **타이머 시작** (휴대폰 90분 또는 `time` 명령).
- 라이브러리 API를 **미리 검색하지 않습니다** — Claude에 맡깁니다.
- 본인이 **코드를 직접 읽고 수정하지 않습니다** — 결과만 보고 자연어로 피드백.
  - 예: "차트가 안 보여" / "에러 메시지 보여줘" / "검은 막대만 나오는데 색깔 다르게 해줘"
- Plan Mode 절대 켜지 않습니다. 4체크 안 합니다.
- 90분이 지나면 **무조건 손을 뗍니다** (완성 여부와 무관).

> 🔥 진짜 90분에 멈추세요. 마음이 급해도 "딱 10분만 더"는 금지 — 그건 다음 챕터에서 살릴 자료입니다.

#### Step 3. 짧은 회고 (15~30분)

작업 디렉토리에 `vibe-notes.md`를 만들고 다음 6가지 질문에 한두 줄씩만 답합니다.

```markdown
# Vibe 90min Notes — <YYYY-MM-DD>

## 만든 것
- 한 줄로:
- 동작 상태: 완성 ✅ / 부분 동작 △ / 미완성 ❌

## 1. 90분 동안 같은 프롬프트를 두 번 이상 친 적?
- 어떤 프롬프트? 왜 다시 쳤나?

## 2. AI가 만든 코드 중 본인이 "이게 진짜 맞나" 확신이 안 섰던 곳?
- 어디서? 왜 의심됐나?

## 3. 가장 답답했던 순간 한 가지
- (예: "Claude가 같은 파일 5번 다시 쓰는데 멈출 줄 모름")

## 4. 가장 즐거웠던 순간 한 가지
- (예: "한 줄 프롬프트로 5초 만에 차트가 나옴")

## 5. 만약 이 결과물을 **외부 사용자**에게 보여줘야 한다면, 지금 상태로 OK?
- 안 된다면 무엇이 빠졌는가? (테스트? 보안? 에러 처리? 다른 사람이 읽을 코드 품질?)

## 6. 같은 일을 다시 한다면 무엇을 다르게 할 것 같나?
- (지금은 답이 막연해도 OK. Ch03 끝나면 명확해질 거예요.)
```

`git add -A && git commit -m "vibe 90min snapshot"` — 이 commit이 **Ch03에서 BEFORE 비교의 시작점**이 됩니다.

### ✅ 산출물
- `~/projects/vibe-90min/` — 90분간 만든 결과물 + `vibe-notes.md`
- 첫 commit 1개 (Ch03 BEFORE 시점)

### 🔍 검증 신호
- 결과물이 **한 가지라도 동작** (불완전해도 OK — 의도적으로 안전망을 뺐기 때문에 동작 안 해도 학습 가치는 있음)
- `vibe-notes.md`의 6개 질문이 **추상적이지 않고 구체** (예: "답답했다"가 아니라 "auth 흐름 디버깅에서 Claude가 같은 가설을 3번 반복")
- Plan Mode·CLAUDE.md·테스트 어느 하나도 **켜지 않은 상태**로 90분 종료
