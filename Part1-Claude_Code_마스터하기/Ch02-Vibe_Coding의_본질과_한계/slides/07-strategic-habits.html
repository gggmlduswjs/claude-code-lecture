전략적 Vibe Coding 3가지 습관

대단한 게 아니라, 시작·진행·확인 단계에 5분씩 더 쓰는 것
1
시작 전에 스펙 쓰기 2~3분
코딩 시작 전 만들 것을 간단히 정리. AI가 추측할 영역을 줄여 일관된 방향을 확보한다.
프로젝트: 할일 관리 앱 핵심 기능: CRUD + 카테고리 분류 + 마감일 알림 기술 스택: Next.js, Supabase, shadcn/ui 제약 조건: 모바일 반응형, 다크 모드 지원
2
한 번에 하나씩 단계 분리
큰 기능을 작은 단위로 쪼개 각 단계에서 방향 확인 + 수정 기회를 만든다.
Step 1: DB 스키마 설계 → 검토 → OK Step 2: API 엔드포인트 구현 → 검토 → OK Step 3: 프론트엔드 UI → 검토 → OK Step 4: 통합 테스트 작성 → 검토 → OK
3
결과 확인 후 피드백 실제 동작
각 단계 종료 시 실제로 실행해보고, 본 것을 기반으로 다음 지시를 만든다.
"서버 실행해줘" → 브라우저에서 확인 → "UI는 좋은데 목록이 비어있어. 초기 데이터 넣어줘"
