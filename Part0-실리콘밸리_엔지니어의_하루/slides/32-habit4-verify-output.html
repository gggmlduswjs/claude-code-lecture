AI 출력 검증 — 4가지 방법

"테스트를 돌려라"는 원칙이 아니라, 구체적인 검증 루프를 돕니다
✅
1. 테스트 실행
Claude에게 "코드에 대한 테스트를 만들고 실행해봐". 실패하면 자동 수정. 생성 → 테스트 → 수정 루프 자체가 검증.
🔍
2. PR 검수
실제 변경된 코드를 PR로 확인. 한 줄씩 볼 필요는 없지만 삭제된 코드는 꼭 확인.
주의: AI가 "더 좋은 방법"이라며 기존 로직을 날려버리는 경우가 있음.
🔄
3. CI/CD 체크 & 개선
테스트 통과했는데 버그가 있다면 테스트를 업데이트. CI가 fail 하면 그 패턴을 분석해서 다시 실수하지 않게 만듭니다.
🤖
4. 서브 에이전트 리뷰
별도 세션으로 띄운 독립된 컨텍스트에서 리뷰. 같은 세션에선 자기 코드에 편향되어 문제를 잘 못 짚습니다.
효과: 엣지 케이스 · 컨벤션 위반 · 보안 허점을 훨씬 잘 잡아냄.
검증에 시간을 아끼지 마세요. 코드 생성 5분 걸렸다고 리뷰도 5분이면 된다는 법은 없습니다.
