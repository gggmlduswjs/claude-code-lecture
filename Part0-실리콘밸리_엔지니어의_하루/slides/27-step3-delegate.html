AI에게 작업 위임

우선순위가 정해지면, 첫 번째 작업을 Claude Code에 넘깁니다
1
이슈 트래커 MCP로 태스크 읽어오기
Linear MCP가 연결되어 있어서, 티켓 번호만 주면 전체 컨텍스트를 가져옵니다.
# 프롬프트 예시
이 이슈를 구현해야 해: "결제 페이지에 할인 코드 입력 필드 추가"
먼저 현재 결제 페이지 코드를 분석하고, 구현 계획을 세워줘. 코드는 아직 쓰지 마.
2
Claude가 관련 파일 탐색 → 계획 제시
CheckoutPage.tsx, PaymentForm.tsx 같은 파일을 찾아 읽고, 기존 패턴 분석 후 1. CouponInput 생성 · 2. PaymentForm 연결 · 3. API 엔드포인트 추가처럼 단계별 계획 출력.
3
위임 후 "관찰하지 않기"
AI가 일하는 동안 나는 미팅에 들어가거나, 다른 작업을 정리합니다. 지켜보면 위임의 의미가 사라집니다. 블로커 발생 시 하네스가 알림.
4
작업 완료 = PR이 이미 올라와 있음
Claude가 브랜치 생성 → 커밋 → PR 올리기까지 자동. PR 본문에 변경사항 요약과 테스트 방법 자동 포함. 하네스가 이 순서를 강제.
AI는 판단을 대신하는 존재가 아니라, 내가 내린 결정을 실행에 옮기는 파트너입니다
