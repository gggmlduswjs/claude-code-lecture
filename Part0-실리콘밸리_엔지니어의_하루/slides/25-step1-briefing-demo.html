실제 터미널에서 브리핑이 돌아가는 과정
1
터미널에서 Claude Code 실행
claude 명령어 한 줄로 세션 시작.
2
Skill로 저장해둔 브리핑 호출
매일 긴 지시를 반복하는 대신 /briefing 한 줄로 실행. Skill 시스템이 반복 프롬프트를 재사용 가능한 자산으로.
3
MCP 도구들이 외부 서비스 호출
화면에 github_list_pull_requests 같은 tool use가 보입니다. Google Calendar MCP · GitHub MCP · Linear MCP 순차 호출.
4
CI 로그 자동 분석
실패한 빌드가 있다면 Claude가 로그를 읽고 "이 테스트가 타임아웃, 원인은 어제 머지된 PR #245의 DB 쿼리 변경"처럼 분석해줍니다.
5
5분 만에 완료
GitHub 웹을 열 필요조차 없이, 터미널 안에서 모든 정보가 들어옵니다. 매일 반복하면 프로젝트의 맥박을 놓치지 않습니다.
