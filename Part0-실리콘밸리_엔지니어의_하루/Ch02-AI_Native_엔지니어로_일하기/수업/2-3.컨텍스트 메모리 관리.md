
클로드 코드를 효과적으로 쓰는 사람들의 공통적인 패턴

1. 짧고 집중된 세션, 하나의 세션=하나의 작업

클로드 코드는 대화가 길어질수록 자동으로 컴팩션을 함 -> 이전 대화를 압축해서 컴팩트하는 과정 -> 이전 대화가 희미해짐 
=>컨텍스트 lost

[/compect]


2. claude.md를 살아있는 문서로 

클로드.md는 클로드 코드가 세션을 시작할 때 자동으로 읽는 매 세션마다 주입되는 컨텍스트 프로젝트 및 설명서이다.
여기에 프로젝트의 아키텍처, 코딩 아키텍처, 빌드 명령어 이런 것들을 적어두면 매 세션마다 반복할 필요도 없고 컨텍스트도 효율적으로 쓸 수 있다. 

간결하게, 살아있게, 매 세션마다 알아야하는 것, 길어지면 참조로 

3. 계획을 검토하는 4가지 체크
- 수정범위가 적절한가
- 기존 패턴을 따르는가
- 테스트 계획이 포함되어 잇던가
- 리스크를 인식하고 있는가

4. ai 출력 검증
- 테스트 실행(생성->테스트->수정)
- pr 검수(실제 변경된 코드를 pr로 확인 - 삭제된 코드는 꼭 확인)
- ci/cd 체크&개선
- 서브 에이전트 리뷰(독립된 컨텍스트에서 리뷰)


5. 메모리 시스템을 잘 이해하고 활용
- 메모리 폴더에 마크다운 파일로 저장
- claude.md -> 팀 전체 규칙, 메모리 -> 개인적
