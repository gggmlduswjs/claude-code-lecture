# Part 0 · Ch02 실습 — AI-Native 엔지니어로 일하기

> **이 챕터의 실습 목적**
> 강사의 하루를 보고 "와 이렇게 일하는구나" 하고 끝나면 의미가 없습니다. **내 하루**를 같은 렌즈로 재매핑하고, 5가지 습관 중 **내가 이번 주에 시작할 2개**를 고른 뒤, 1주일 트래커로 실제 변화를 측정합니다.

**예상 소요 시간**: 30분 (자가진단 15분 + 트래커 셋업 15분) + **1주일 백그라운드 작업** (매일 1~2분 체크)

**준비물**
- Ch01에서 만든 `my-level.md` (강의 실습 repo 본인 fork)

## 실습 1 — 내 하루를 "수집·판단·실행" 렌즈로 재매핑 (15분)

### 무엇을 하는가
강의에서 본 강사의 하루(09:00 브리핑 → 분류·위임 → 14:00 검수 → 17:00 cron)를 본인 하루에 그대로 복붙할 필요는 없습니다. 핵심 루프는 **수집(AI) → 판단(사람) → 실행(AI)** — 이걸 본인 하루의 시간 블록에 매핑하는 게 진짜 실습입니다.

### 단계

1. fork한 강의 실습 repo의 이번 챕터 폴더로 이동.
   ```bash
   cd claude-code-lecture/Part0-실리콘밸리_엔지니어의_하루/Ch02-AI_Native_엔지니어로_일하기/exercises/templates
   ```

2. `my-workflow.md`를 **직접 열어서** 본인 답으로 채웁니다. (본인 fork이므로 별도 복사 없이 그대로 commit.)

3. 아래 3개 섹션을 채웁니다.

#### Section A. 어제(혹은 지난 평일) 하루 시간 블록
대략적으로 30분 단위로 적습니다. 예:
```
09:00–09:30  메일 확인
09:30–11:00  팀 미팅
11:00–12:30  Jira 티켓 작업 (인증 미들웨어)
...
```

#### Section B. 각 블록에 "수집 / 판단 / 실행" 태그
모든 블록 옆에 다음 중 하나를 적습니다 — 또는 복수 태그.
- 🔵 **수집** (정보 모으기 — 이슈/PR/지표/문서 읽기)
- 🟡 **판단** (사람의 결정 — 우선순위, 설계, 리뷰 승인/거절)
- 🔴 **실행** (코드/문서/PR 만들기)

#### Section C. AI 위임 후보
판단 ❌ + (수집 또는 실행) ✅ 인 블록 **3개**를 골라, "이 블록을 AI에 어떻게 위임할지" 한 문장씩 적습니다.

예시:
```
- 09:00–09:30 메일 확인 (🔵수집)
  → /briefing 스킬 또는 cron으로 매일 아침 자동 요약 받기
- 14:00–14:30 PR 리뷰 (🟡판단 + 🔵수집)
  → 수집(diff/테스트결과/CI상태)은 AI에 자동화, 승인 결정만 내가
- 16:00–17:00 단위 테스트 작성 (🔴실행)
  → 유닛 테스트 작성 서브에이전트에 위임 (대상 함수만 지정, 테스트 케이스는 에이전트가 도출)
```

### ✅ 산출물
- `my-workflow.md` — Section A·B·C 완성. git commit 권장.

### 🔍 검증 신호
- 시간 블록이 어제 하루를 70% 이상 덮음 (회의·메일·코딩·디버깅·잡무)
- C 섹션의 3개 블록이 "정확히 어떻게 위임할지"까지 구체적임 (도구 이름 또는 서브에이전트 이름 언급)

## 실습 2 — 5가지 습관 중 2개 선택 + 1주일 트래커 (15분 셋업 + 1주일 진행)

### 무엇을 하는가
강의에서 본 5가지 습관(짧은 세션 · CLAUDE.md 생활화 · 계획 검토 4체크 · 다중 검증 · Memory)을 한 번에 다 시작하면 다 잊혀집니다. **2개만 골라**, 1주일 동안 매일 체크합니다.

### 단계

1. 같은 폴더의 `habit-tracker.md`를 엽니다 — `my-workflow.md` 바로 옆에 있습니다.
2. 5개 습관 중 **2개**에 ⭐ 표시. 나머지 3개는 다음 주 후보로 남깁니다.
3. 매일 저녁 (또는 다음 날 아침) **1~2분** 체크. 7일 채우는 것이 목표.

#### 5가지 습관 (강의 복습)

| 습관 | 무엇 | 한 번에 가능한가? |
|---|---|---|
| 짧은 세션 | 한 세션 = 한 작업, 컨텍스트 30~40% 유지 | 즉시 시작 가능 ⭐ 후보 |
| CLAUDE.md 생활화 | 새 컨벤션·결정 → 즉시 CLAUDE.md에 반영 | Part 1 Ch01 직후가 더 좋음 |
| 계획 검토 4체크 | 범위·패턴·테스트·리스크 4항목 매번 확인 | 즉시 시작 가능 ⭐ 후보 |
| 다중 검증 | 테스트 + PR 검수 + CI + 서브에이전트 리뷰 | Part 1 Ch01 직후가 더 좋음 |
| Memory 활용 | 개인 선호도를 `/memory`에 저장, 매주 정리 | 즉시 시작 가능 ⭐ 후보 |

> 추천: 처음에는 **짧은 세션 + 계획 검토 4체크** 두 가지부터.

### ✅ 산출물
- `habit-tracker.md` — 7일 동안 매일 체크. 1주 후 마지막에 회고 한 문단.

### 🔍 검증 신호
- 7일 중 5일 이상 체크됨 (완벽 아니어도 OK, "기록" 자체가 신호)
- 7일째 회고에 "이번 주 패턴" 1개 발견 (예: "오후에 컨텍스트가 항상 70% 넘는 걸 발견했다")
