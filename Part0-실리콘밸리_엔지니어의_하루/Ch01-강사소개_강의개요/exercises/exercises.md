## 실습 — 현재 나의 AI 코딩 레벨 자가진단 (10분)

### 무엇을 하는가
이 강의는 **Vibe Coding · Agentic Engineering · Harness Engineering 세 가지 모두**를 다룹니다. 어느 한쪽을 우월하게 두는 게 아니라, **각 상황에서 어떤 모드로 일할지 본인이 고를 수 있게** 만드는 것이 목표입니다.

그 출발점을 찾기 위해, 현재 본인이 AI 코딩 도구를 어떻게 쓰고 있는지 한 페이지로 정리하세요.

### 단계

1. 강의 실습 repo를 **fork**합니다 → https://github.com/jha0313/claude-code-lecture
   ```bash
   # fork 후 본인 계정의 repo를 clone
   git clone https://github.com/<your-username>/claude-code-lecture.git
   cd claude-code-lecture/Part0-실리콘밸리_엔지니어의_하루/Ch01-강사소개_강의개요/exercises/templates
   ```

2. `templates/my-level.md`를 **직접 열어서** 본인 답으로 채웁니다. (별도 복사 불필요 — 본인 fork이므로 그대로 commit하면 됩니다.)

3. `my-level.md`의 표 7개 항목에 ✅ / ❌ / △ 로 표시합니다.

| 항목 | 현재 나 | 비고 |
|---|---|---|
| Claude Code(또는 Cursor/Copilot)를 매일 코딩에 쓰고 있다 | | |
| AI에 작업을 줄 때 "어떻게 검증할지"를 미리 정의한다 | | |
| 같은 프롬프트를 두 번 이상 반복해서 친 적이 있다 | | |
| 프로젝트에 `CLAUDE.md` 또는 그에 준하는 AI용 가이드 문서가 있다 | | |
| AI가 만든 코드를 PR로 올린 적이 있다 | | |
| AI를 위해 만든 Skill / Hook / 자동화 스크립트가 1개라도 있다 | | |
| AI가 자기 결과를 스스로 검증(테스트·로그)하도록 만든 적이 있다 | | |

4. 표 아래에 한 문장만 직접 적습니다.
   - "이 강의가 끝났을 때 (___)을(를) 할 수 있게 되고 싶습니다."

### ✅ 산출물
- `my-level.md` — 본인 작업 공간에 commit 권장 (5주 후 다시 보고 변화 측정)

### 🔍 검증 신호
- 표 7개 항목이 모두 채워졌다.
- 목표 문장이 25자 이상 구체적이다 (예: "회사 repo에 CLAUDE.md를 도입해 PR 리뷰 시간을 절반으로 줄인다").
